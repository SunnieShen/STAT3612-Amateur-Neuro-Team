# STAT3612 Group Project Proposal: Multimodal Presurgical Brain Tumor Classification

**Group Members**: [Member 1], [Member 2], [Member 3], [Member 4], [Member 5]

## 1. Project Overview

This project develops statistical machine learning models for presurgical brain tumor classification into five categories — Glioma, Meningioma, Brain Metastase Tumour, Tumors of the Sellar Region, and Pineal Tumour and Choroid Plexus Tumour — using multimodal clinical data. Our approach follows a **three-stage pipeline**: (1) single-modality baselines with cross-validation to understand each modality's contribution, (2) multimodal fusion comparing early and late strategies with comprehensive evaluation, and (3) optimization through class imbalance mitigation, feature selection, and interpretability analysis.

We leverage all available data modalities: (a) deep image features extracted from ResNet (2048-d per MRI sequence, stored in `image_features/`), (b) radiomic features extracted via PyRadiomics (5 features × 4 MRI sequences, in `radiomics_info/`), (c) structured clinical information including patient demographics (age, gender) and report-derived fields (tumor location, signal intensity descriptions for T1/T1c/T2/FLAIR, in `clinical_information/`), and (d) raw radiology reports describing MRI findings (in `original_raw_report/`). The Kaggle competition evaluates using **macro-averaged F1**; given the severe class imbalance (~40:1 between the largest and smallest classes), this is also our **primary evaluation metric**, supplemented by weighted-F1, accuracy, per-class F1, and macro-averaged AUROC. The dataset contains 1,983 training / 283 validation / 572 test patients.

**Evaluation methodology**: All model evaluation follows a two-step protocol grounded in **Lecture 4.1** (Cross-Validation): (1) **stratified 5-fold CV on the training set** as the primary model selection criterion, reporting mean ± std of macro-F1 across folds, and (2) **held-out validation set** (`val.json`, 283 patients) as a reference confirmation. This ensures robust performance estimates despite the small sample size and severe class imbalance.

## 2. Preliminary Findings

Initial EDA, single-modality baselines, and a full multimodal fusion experiment have been completed:

- **Class imbalance**: Glioma (924, 46.6%) and Meningioma (728, 36.7%) dominate, while Pineal/Choroid Plexus (23, 1.2%) and Sellar Region (56, 2.8%) are severely underrepresented. We adopt `class_weight='balanced'` as the default mitigation throughout all experiments.
- **Multimodal incompleteness**: ~20% of cases lack one MRI modality; we zero-pad missing feature vectors as structural missingness.
- **Missing demographics**: JSON metadata has ~70% missing for Age/Sex; we use `clinical_information` CSVs (fully populated) as the single source of truth.
- **Single-modality baselines** (LightGBM, 5-fold CV on training set): Text features (TF-IDF 500-d) yield the highest CV macro-F1, followed by clinical features (24-d), confirming that structured clinical data and radiology report text carry the strongest classification signal. Image features (PCA-256) and radiomics (20-d) alone perform substantially lower.
- **Multimodal fusion result**: Early fusion (800-d concatenated features) with LightGBM achieves the best held-out macro-F1 among single-model approaches. Soft Voting and Stacking ensembles provide additional marginal gains.
- **Modality ablation**: Fusing all four modalities outperforms any single modality or partial combination, confirming the complementary value of each data source.
- **Key insight**: Class weighting alone does not sufficiently lift minority-class F1 (Pineal/Choroid Plexus and Sellar Region remain near-zero F1); the high dimensionality (800-d vs 1,983 samples) creates overfitting risk requiring regularization and feature selection.

## 3. Methods

### Evaluation Framework (Lecture 4.1)

All experiments use a consistent two-step evaluation protocol:

1. **Stratified 5-fold CV on training set** (primary): The training set (1,983 patients) is split into 5 stratified folds. For each fold, we build a `Pipeline(StandardScaler, Model)` to prevent data leakage, train on 4 folds, and evaluate on the held-out fold. We report the mean ± standard deviation of macro-F1 across all 5 folds. This is the metric used for **model selection**.
2. **Held-out validation** (reference): The selected model is retrained on the full training set and evaluated on the official `val.json` (283 patients). We report accuracy, macro-F1, weighted-F1, per-class precision/recall/F1, and a confusion matrix. This serves as a final confirmation.

> **Why stratified?** With ~40:1 class imbalance, random folds could leave zero samples of the rarest class in some folds. Stratification guarantees proportional class representation in every fold.

### Stage 1: Single-Modality Baselines

Before fusion, we build independent classifiers for each modality using LightGBM (`class_weight='balanced'`) to understand their individual discriminative power under a unified model:

- **Deep image features (8192-d → PCA-256)**: 4 MRI sequence features concatenated after zero-padding for missing modalities, reduced via PCA to 256 dimensions.
- **Radiomics (20-d)**: 5 radiomic features × 4 MRI sequences (mean, entropy, 90th percentile, GLCM contrast, GLCM joint entropy), with NaN → 0 imputation.
- **Clinical features (24-d)**: Encoded demographics (age, gender), signal intensity descriptors, and tumor location keywords from `clinical_information/` CSVs.
- **Text features (TF-IDF 500-d)**: Unigram TF-IDF vectors extracted from radiology reports in the JSON metadata.

Each modality is evaluated via 5-fold CV (primary) and held-out validation (reference), with full classification reports and confusion matrices. Results are consolidated in a summary heatmap and per-class F1 grouped bar chart.

### Stage 2: Multimodal Fusion

All four modality feature vectors are concatenated after standardization, forming an **800-d unified feature vector**. We compare two fusion strategies:

**Early fusion** — train a single model on the concatenated 800-d space. Six classifiers of increasing flexibility are compared, each evaluated with integrated 5-fold CV:

| Model | Course Reference | Rationale |
|---|---|---|
| Logistic Regression (L2) | Lecture 4.2 | Linear baseline; regularization controls overfitting in 800-d space |
| Random Forest | Lecture 5.1 | Bagging-based ensemble reducing variance via bootstrap aggregation |
| SVM-RBF | Lecture 7 | Kernel trick for non-linear separation in high-dimensional space |
| XGBoost | Lecture 5.2 | Boosting ensemble reducing bias by focusing on misclassified examples |
| LightGBM | Lecture 5.2 | Efficient gradient boosting with leaf-wise growth; supports `class_weight` natively |
| MLP | — | Non-linear neural network baseline (3 hidden layers: 512→256→128) |

**Late fusion** — combine predictions from individually trained models:

- **Soft Voting**: Weighted probability averaging over SVM-RBF, XGBoost, LightGBM, and LR.
- **Stacking**: A Logistic Regression meta-learner trained on base model outputs via internal 5-fold CV (SVM-RBF, XGBoost, LightGBM, RF as base learners).

All models are compared in a comprehensive result table (accuracy, macro-F1, weighted-F1, macro AUROC), confusion matrix grid, multiclass ROC curves (one-vs-rest, macro-averaged), and precision–recall curves.

### Stage 3: Optimization and Class Imbalance

- **Feature selection**: Compare LASSO-based sparsity (L1 LogisticRegression), tree-based importance (RF and LightGBM top-k features), and Mutual Information ranking to identify the most informative feature subset. A modality-source breakdown of top features is reported.
- **PCA tuning**: Evaluate the trade-off between explained variance and downstream macro-F1 for the image block (128 / 256 / 512 components).
- **Class imbalance**: Controlled comparison of (a) `class_weight='balanced'` only (current default), (b) random oversampling of minority classes (train only), and (c) SMOTE applied in the PCA-reduced feature space (train only). Evaluated by per-class F1 and macro-F1 with focus on the two rarest classes (Pineal/Choroid Plexus and Sellar Region).

## 4. Model Analysis Plan

Targeting the workload and analysis component (40% of grade):

- **Model comparison table**: Compare all candidate models (6 early fusion + 2 late fusion) using 5-fold CV macro-F1 (primary), held-out macro-F1, accuracy, weighted-F1, and macro-averaged AUROC, with results summarized in a single heatmap table.
- **Modality ablation study**: Systematically remove each modality from the early fusion vector and measure the macro-F1 drop (using LightGBM), quantifying each modality's marginal contribution. We also test all pairwise modality combinations.
- **Early vs late fusion comparison**: Determine which fusion strategy better leverages cross-modality information, analyzing whether late fusion (which preserves modality-specific structure) outperforms early fusion (which allows cross-modality interactions).
- **Multiclass ROC and PR curves**: Plot one-vs-rest macro-averaged ROC and precision–recall curves for all models to visualize discrimination ability beyond the single F1 threshold.
- **Feature importance**: Compare Random Forest Gini importance and LightGBM split-count importance on the 800-d fused feature space, with a modality-source breakdown (how many of top-30 features come from image / radiomics / clinical / text).
- **Class-wise error analysis**: Confusion matrix of the best model to identify the most confused class pairs; investigate which modalities/features differentiate them. Report top-3 off-diagonal misclassification patterns.
- **Interpretability** *(planned)*: SHAP values [3] on the best tree-based model to provide instance-level explanations and contrast with LASSO coefficients for global feature ranking.
- **Imbalance strategy comparison**: Report the effect of each resampling strategy on minority-class F1 and overall macro-F1.
